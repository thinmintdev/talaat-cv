Tailwind is a great low-level tool. Architecture makes it a great system.

## My rules of thumb

### 1) Co-locate intent with components
- Create lightweight wrappers: `Button`, `Card`, `Badge`
- Use `@apply` sparingly—prefer composition

### 2) Design tokens > ad-hoc colors
- Define palette, radii, spacing once
- Expose via `theme.extend`

### 3) Variants with class-variance-authority
- Encode state (size, intent, layout) without props explosion

```ts
import { cva } from 'class-variance-authority'

export const button = cva(
  'inline-flex items-center justify-center rounded-md font-medium transition-colors',
  {
    variants: {
      intent: {
        primary: 'bg-blue-600 text-white hover:bg-blue-700',
        ghost: 'bg-transparent hover:bg-gray-100',
      },
      size: {
        sm: 'h-8 px-3 text-sm',
        md: 'h-10 px-4',
        lg: 'h-12 px-6 text-lg',
      },
    },
    defaultVariants: { intent: 'primary', size: 'md' },
  }
)
```

### 4) The 80/20
- 80% utilities, 20% components
- No more than ~10 custom CSS files in most apps
